# 🟩 扫描器规格与 Liberation

### 扫描器规格的现实情况

点率和扫描器规格有时会让人困惑。你经常会看到类似 30kpps @ 8º 或 50kpps @ 4º 的规格，但这些数字实际代表什么并不总是很明显。

{% hint style="info" %}
免责声明：我不是扫描器硬件专家，但我有多年通过软件和点流生成，而不是硬件调校，让不同质量扫描器获得良好效果的实践经验。本文基于这些经验。
{% endhint %}

#### **这些数字从哪里来**

“30K”和“50K”这类说法，是基于在特定条件下用 ILDA 测试图案、以对应点率评估扫描器的简写。

当一台扫描器标注为类似 _30Kpps @ 8°_ 时，它真正的意思是：

> “这台扫描器在正确调校后，可以以 30,000 点/秒、8° 扫描角重现 ILDA 测试图案。”

这不是一个全面或完全标准化的真实世界性能测量。事实上，它最初并不是作为基准测试设计的，而是用于**调校流程**。你以固定点率（例如 30,000 点/秒）运行一个已知图案，然后调整阻尼和增益，直到图案看起来正确。

但它仍然是目前最常用的参考，至少对于信誉良好的制造商来说，可以大致判断扫描器质量。不过对于_不那么可靠_的制造商……

#### 如果你想按标称规格测试扫描器

{% hint style="danger" %}
**这是高级技巧，如果不小心可能损坏扫描器。不建议使用，除非你明确知道自己在做什么。**
{% endhint %}

你需要找到可以输出 [ILDA Test Pattern](https://ilda.com/technical.htm?r=7950) 的软件——我认为 LaserShowGen 可能可以做到——并将输出尺寸调整到指定扫描角（例如 8°）。请参考 ILDA 文档了解如何分析输出。

#### 为什么它可能不是好的基准

首先，即使扫描器本身不错，如果它们的调校方式并不是针对该测试图案优化的，测试图案也可能显示不正确。

它可以作为区分“好”和“差”扫描器的通用参考，但制造商有时会比较随意地使用这些规格。

#### 那我该如何选择好的扫描器？

我认为最重要的是确认它们来自可靠制造商。Cambridge Technology（CT）、Eye Magic（EMS）和 ScannerMAX（Pangolin 子公司）这类昂贵的高端扫描器制造商通常都很好，基本不会出错。但一对扫描器价格接近 1000 美元时，对许多刚起步的人来说，甚至比激光本身还贵！

所以我大多使用中国制造商的产品。Dragon Tiger（DT）扫描器价格合理、表现不错，我认为 LightSpace 也使用它们。许多其他制造商（包括某些 OPT 和 Able 型号）也使用基于 DT 的系统。

Phenix Technology（PT）通常低一个档次，但说实话，对大多数用途可能也够用。

**如果你的扫描器没有品牌，那才是你真正需要担心质量的时候！**

#### Liberation 如何帮助

首先，对大多数事情来说，你并不需要非常昂贵的扫描器！价格合理的 30kpps DT，甚至 PT，通常都可以胜任。默认扫描器设置有意较为保守，在大多数情况下_你不需要调整它们_（除了 _Scanner sync_）。

即使你有更好的扫描器，也没必要让它们比实际需要更吃力地工作。这会显著延长它们的寿命。

#### 什么是“点流”？

你可能以前听过这个词，它指的是我们控制扫描器路径的方式。

点流是一组 X/Y 位置列表，每个位置都带有颜色。要绘制一条白线，你会沿着那条线发送一系列点，并将它们都设为白色。扫描器随后以固定速率，也就是每秒点数（PPS），从一个点移动到下一个点，光束就会描绘出这个形状。

#### 传统激光软件中的工作方式

传统激光软件会为每个 cue 存储一段点流。对于动画 cue，通常意味着每一帧都有独立的点流。关键在于，一切都是完全预先确定的。因此，提高点率会让扫描器以更快速度通过同一组预定义数据。

{% hint style="info" %}
对于较旧的软件来说，这种方式是必要的，因为当时电脑还不够快，无法实时生成点流。这也是为什么通常会有单独的 cue 编辑器，用于预生成每一帧动画的数据。

这也解释了为什么内容会占用数 GB 空间。你实际上是在以相当高的采样率存储大量未压缩波形。
{% endhint %}

#### 为什么“点率”在 Liberation 中意义较小

Liberation 会实时生成点流，这给了我们极大的灵活性。注意 Laser Settings 面板中的 “Scanner speed” 设置。它让我们加快或减慢扫描器速度，但重要的是，它**不会**改变底层点率（PPS）。

#### 等等，为什么？

我知道，乍看起来有点奇怪。

因为 Liberation 实时生成点流，所以它可以调整这个点流。点之间分布得越开，扫描器移动越快。点之间越密，扫描器移动越慢。

{% hint style="info" %}
在近期版本的 Liberation 中，高级设置里的实际 _point rate_ 完全不会影响扫描器速度。渲染器会根据所选点率调整点的分布，同时保持扫描器运动不变。

这确实会影响点路径的“分辨率”，但主要只在图形内容中产生差异（也有助于更精细地调整 _scanner sync_ 设置）。
{% endhint %}

#### 很好！那这到底意味着什么？

好问题。我的建议是：

* 避免无品牌扫描器。如果能买到更快的扫描器，就选更快的，最低建议 30KPPS。
* 对大多数情况来说，DT30 扫描器足够好，便宜激光里的 PT30 也可能可以。
* 如果你做图形内容，大多数情况下更多激光会比更快扫描器更有帮助。
* 到了更高端的配置，任何成熟高端品牌通常都没问题。
* 如果你只能买到最便宜的无品牌扫描器，Liberation 的默认设置相当保守，基础 beam 内容通常也能有还可以的效果。如果效果吃力，请降低 **Speed** 设置（但不要改点率！）。

#### 那 ILDA 测试图案呢？

……它作为校准和参考工具仍然非常有用，但它从来不是为了成为全面基准测试而设计的，也可能被制造商误用或宽松解读。
