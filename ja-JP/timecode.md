# ✅ Timecode

Liberation は、外部の SMPTE/LTC タイムコード信号との同期に対応しています。これはライブ音楽ショーで、ライト、パイロ、映像、バッキングトラックのタイミングを揃えるためによく使われます。

{% hint style="info" %}
SMPTE/LTC とは？

SMPTE はタイムコードの標準規格で、LTC はこのタイムコードを音声信号に変換したものです。この音声を聞くと、不快な高音のきしみのように聞こえますが、コンピューターにとっては高解像度のタイミング情報です。

**豆知識！**

歴史的に、SMPTE は映像と音声の同期に使われてきました。また、アナログテープと同期する場合は、テープの 1 トラックにタイムコード音声を録音していました。これはテープを「ストライピング」すると呼ばれることもあります。このタイムコードトラックを使うことで、複数のテープデッキ同士のタイミングを合わせたり、MIDI シーケンサーをテープに同期させたりできました。（つまり、MIDI 楽器をテープに録音しなくても、ミックス中にシーケンサーでライブ再生させることができたわけです！）

SMPTE は、この標準を定義した Society of Motion Picture and Television Engineers の略です。LTC は _Linear TimeCode_ の略です。
{% endhint %}

LTC タイムコード信号は、コンピューター上の任意のオーディオインターフェースで受信できます。ただし、少なくとも 1 つの調整可能な XLR 入力とモニタリング機能を備えたプロ向けインターフェースの使用をおすすめします。

[M-Audio 192/6](https://www.m-audio.com/audio-midi-interfaces/air-192-6.html) は、ヘッドホンモニタリング、2 つの XLR 入力があり、特別なドライバーも不要（少なくとも macOS では）なので、良い使用感でした。タイムコード専用でしか使わない場合は、少し安い [M-Audio 192/4](https://www.m-audio.com/audio-midi-interfaces/air-192-4.html)（入力は 1 つのみで MIDI なし）でも構いませんが、正直なところ、ある程度まともなオーディオインターフェースであれば問題なく動作するはずです。

{% hint style="info" %}
LTC タイムコード信号は通常、バランス XLR ケーブルで配信されます。これは、長距離でも低レベルの音声信号を安定して伝送できるためです。（XLR は、通常マイクで使われる丸型のジャックコネクターです）
{% endhint %}

### ハードウェア接続

タイムコード信号の XLR ケーブルをオーディオインターフェースに接続し、良好な信号が入っていることを確認します。信号レベルが十分に強く、かつクリッピングしないように、オーディオインターフェース側でレベルを調整します。オーディオインターフェースにヘッドホンジャックがある場合は、タイムコードを実際に聞いて、グリッチがないこと、ノイズが多すぎないことを確認できます。

{% hint style="info" %}
理論上は、MacBook のジャック端子から信号を受信することも可能ですが、その場合はカスタムケーブルが必要になります。これらのジャックは通常 4 極 TRRS ミニジャックで、正直なところ、どの端子をオーディオ入力として使えるのか、また対応できる電圧レベルがどの程度なのかも確信がありません（どこかで +/-1V と読んだことがありますが、使用する場合は自己責任でお願いします！）

これを試すより、安価な USB オーディオインターフェースを用意する方が良いと思います。
{% endhint %}

オーディオインターフェースに入力モニタリング機能がない場合は、macOS のシステム設定（_Sound_ 内）で信号が入っていることを確認できます。（Windows では _Sound Control Panel_ を使用します）。

### Liberation での設定

1. Timecode settings Window で、オーディオインターフェースと正しい入力チャンネルを選択します。

![](.gitbook/assets/timecode-panel.png)

{% hint style="info" %}
オーディオインターフェースの各入力チャンネルごとに、ドロップダウンメニュー内に個別の選択肢が用意されている点に注意してください。

<img src=".gitbook/assets/m-audio.png" alt="" data-size="original">
{% endhint %}

左側の四角に注目してください。有効なタイムコード信号を受信している場合は緑になります。受信できていない場合は赤になります。

2. 入力されるタイムコードに対して正しいフレームレートを選択します。タイムコードを提供する担当者に確認すれば、フレームレートを教えてもらえるはずです。（間違っていても同期はしますが、毎秒小さな「スキップ」が発生するのに気づくはずです）
3. タイムラインバー上の小さな時計アイコンを使って Timeline のタイムコード設定を開き、SMPTE(LTC) オプションを選択します。

<figure><img src=".gitbook/assets/timeline-settings.png" alt=""><figcaption></figcaption></figure>

4. 曲の開始位置に合わせて、開始オフセット（時、分、秒、フレーム）を調整します。複数のタイムラインがある場合は、それぞれ個別にこれらのオプションを設定する必要があります。

{% hint style="info" %}
ツアーの現場では、各曲を異なる時刻から開始するのが一般的な慣例です。たとえば 1 曲目は 01:00:00:00、2 曲目は 02:00:00:00、というように設定します。

Liberation はタイムコードに応じて自動的にタイムラインを切り替えるため、ショー中に手動でタイムラインを変更する必要はありません。
{% endhint %}

MIDI Clock や Ableton Link とは異なり、SMPTE は時、分、秒、フレームで計測される _絶対_ 時間システムです。Liberation のコア時間システムはビートと小節を基準にしているため、タイムコードを受信すると、タイムラインで設定されたテンポが使用されます。このテンポが、タイムコードに同期している音楽と一致していることを確認する必要があります。
