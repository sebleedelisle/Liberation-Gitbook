# 🟩 スキャナー仕様と Liberation

### スキャナー仕様のややこしい現実

ポイントレートやスキャナー仕様は、少し分かりにくいことがあります。30kpps @ 8º や 50kpps @ 4º のような仕様をよく見かけますが、その数字が実際に何を表しているのかは、必ずしも明確ではありません。

{% hint style="info" %}
免責事項 - 私はスキャナーハードウェアの専門家ではありません。ただし、ハードウェア調整ではなく、ソフトウェアとポイントストリーム生成によって、さまざまな品質のスキャナーを見栄えよく動作させてきた実務経験が何年もあります。ここでの内容は、その経験に基づいています。
{% endhint %}

#### **これらの数値はどこから来ているのか**

「30K」や「50K」といった表現は、特定の条件下で、そのポイントレートにおいて ILDA テストパターンを使ってスキャナーを評価する方法に基づいた略称です。

たとえばスキャナーが _30Kpps @ 8°_ と記載されている場合、実際には次のような意味です。

> 「このスキャナーは、適切に調整されていれば、8° のスキャン角で毎秒 30,000 ポイントの ILDA テストパターンを再現できます。」

これは、実運用での性能を包括的かつ完全に標準化して測定したものではありません。実際、もともとベンチマークとして設計されたものでもなく、**調整手順**のために使われていました。既知のパターンを固定ポイントレート（例: 毎秒 30,000 ポイント）で出力し、正しく見えるまでダンピングとゲインを調整します。

とはいえ、現在でも最も広く使われている基準であり、少なくとも信頼できるメーカーのスキャナーであれば、その品質を把握するよい目安になります。ただし、_あまり信頼できない_ メーカーの場合は……

#### 定格どおりにスキャナーをテストしたい場合

{% hint style="danger" %}
**これは高度なテクニックであり、注意しないとスキャナーを損傷する可能性があります。自分が何をしているか理解している場合を除き、推奨しません。**
{% endhint %}

[ILDA Test Pattern](https://ilda.com/technical.htm?r=7950) を出力できるソフトウェアを探す必要があります。LaserShowGen ならできるかもしれません。そして、指定されたスキャン角（例: 8°）に合わせて出力サイズを調整します。出力の解析方法については、ILDA のドキュメントを参照してください。

#### なぜ良いベンチマークとは限らないのか

まず、スキャナー自体が良好でも、そのテストに最適化された調整になっていなければ、テストパターンが正しく表示されないことがあります。

「良い」スキャナーと「悪い」スキャナーを見分ける一般的な目安としては役立ちますが、メーカーによっては、これらの仕様をかなり大まかに扱うことがあります。

#### では、どうやって良いスキャナーを選べばよいのか

基本的には、信頼できるメーカー製であることを確認すればよいと思います。Cambridge Technology (CT)、Eye Magic (EMS)、ScannerMAX（Pangolin の子会社）のような高価なハイエンドスキャナーメーカーは常に信頼でき、まず間違いありません。ただし、スキャナーのペアが約 1000 ドルになると、始めたばかりの多くの人にとっては、レーザー本体より高価になってしまいます。

そのため、私は主に中国メーカーのものを使ってきました。Dragon Tiger (DT) のスキャナーは手頃で十分な品質があり、LightSpace もこれを使っていると思います。他の多くのメーカー（OPT や Able の一部モデルを含む）も、DT ベースのシステムを使っています。

Phenix Technology (PT) は一般的には一段低いクラスですが、正直なところ、ほとんどの用途では十分だと思います。

**スキャナーがノーブランドの場合は、そのときこそ品質を心配したほうがよいでしょう。**

#### Liberation がどのように役立つか

まず、ほとんどの用途では本当に高価なスキャナーは必要ありません。手頃な 30kpps の DT、場合によっては PT でも十分です。デフォルトのスキャナー設定は意図的に控えめにしてあり、ほとんどの場合、_調整する必要はありません_（_Scanner sync_ を除く）。

より良いスキャナーを持っている場合でも、必要以上に強く駆動する意味はありません。そうすることで、寿命を大きく延ばせます。

#### 「ポイントストリーム」とは何か

この用語はおそらく聞いたことがあると思います。これは、スキャナーの軌跡を制御する方法です。

ポイントストリームとは、X/Y 位置のリストであり、それぞれに色が付いています。たとえば白い線を描く場合、その線に沿って白に設定した一連のポイントを送ります。スキャナーは、固定レート、つまり PPS（ポイント毎秒）でポイントからポイントへ移動し、ビームが形状を描きます。

#### 従来のレーザーソフトウェアでの仕組み

従来のレーザーソフトウェアでは、各キューごとにポイントストリームを保存します。アニメーションキューの場合、通常は各フレームごとに個別のポイントストリームを持ちます。重要なのは、すべてが完全に事前決定されているという点です。そのため、ポイントレートを上げると、同じ事前定義済みデータをスキャナーがより速く移動することになります。

{% hint style="info" %}
古いソフトウェアでは、この方式が必要でした。当時のコンピューターは、リアルタイムでポイントストリームを生成できるほど高速ではなかったためです。そのため通常は、アニメーションの各フレーム用データを事前生成するための別個のキューエディターがあります。

また、コンテンツがギガバイト単位の容量を占める理由もこれで説明できます。実質的には、高いサンプルレートの大きな非圧縮波形を保存しているようなものです。
{% endhint %}

#### Liberation では「ポイントレート」の意味が薄い理由

Liberation はポイントストリームをリアルタイムで生成するため、非常に高い柔軟性があります。Laser Settings パネルの "Scanner speed" 設定に注目してください。これによりスキャナーを速くしたり遅くしたりできますが、重要なのは、基礎となるポイントレート（PPS）は**変更されない**という点です。

#### えっ、どういうこと？

最初は奇妙に聞こえると思います。

Liberation はポイントストリームをリアルタイムで生成しているため、そのポイントストリームを調整できます。ポイント同士の間隔が広いほど、スキャナーは速く動きます。ポイント同士が近いほど、スキャナーは遅く動きます。

{% hint style="info" %}
最近の Liberation では、実際の _point rate_（advanced settings 内）はスキャナー速度にまったく影響しません。代わりに、レンダラーが選択されたポイントレートに合わせてポイント分布を調整しつつ、スキャナーの動きは変えないようにしています。

これはポイントパスの「解像度」には影響しますが、その違いが主に関係するのはグラフィックスの場合です（また、_scanner sync_ 設定をより細かく調整する助けにもなります）。
{% endhint %}

#### なるほど。では実際には何を意味するのか

よい質問です。私からのアドバイスは次のとおりです。

* ノーブランドのスキャナーは避けてください。より高速なスキャナーを入手できるなら、そうしてください。最低でも 30KPPS です。
* ほとんどの場合、DT30 スキャナーで十分です。安価なレーザーでは、PT30 スキャナーでもおそらく問題ありません。
* グラフィックスを行う場合、ほとんどのケースでは、より高速なスキャナーよりもレーザー台数を増やすほうが効果的です。
* ハイエンド構成になってくると、定評のあるハイエンドブランドであればどれでも問題ありません。
* 最も安価なノーブランドスキャナーしか入手できない場合でも、Liberation のデフォルト設定はかなり控えめなので、基本的なビーム用途であればおそらく問題ない結果が得られます。厳しい場合は **Speed** 設定を下げてください（ただし、ポイントレートは変更しないでください）。

#### では ILDA Test Pattern は？

……今でもキャリブレーションや基準ツールとして非常に有用です。ただし、包括的なベンチマークとして設計されたものではなく、メーカーによって誤用されたり、ゆるく解釈されたりすることがあります。
