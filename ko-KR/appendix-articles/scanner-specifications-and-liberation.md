# 🟩 Scanner 사양과 Liberation

### Scanner 사양의 복잡한 현실

포인트 레이트와 Scanner 사양은 조금 헷갈릴 수 있습니다. 30kpps @ 8º 또는 50kpps @ 4º 같은 사양을 자주 보게 되지만, 이 숫자들이 실제로 무엇을 의미하는지는 항상 명확하지 않습니다.

{% hint style="info" %}
고지 - 저는 Scanner 하드웨어 전문가는 아니지만, 하드웨어 튜닝이 아니라 소프트웨어와 포인트 스트림 생성 방식으로 다양한 품질의 Scanner를 보기 좋게 만드는 실무 경험을 여러 해 쌓아 왔습니다. 아래 내용은 그 경험을 바탕으로 합니다.
{% endhint %}

#### **이 숫자들은 어디서 나오는가**

“30K”, “50K” 같은 용어는 특정 조건에서 해당 포인트 레이트로 ILDA 테스트 패턴을 사용해 Scanner를 평가하는 방식에서 나온 약칭입니다.

예를 들어 Scanner 사양이 _30Kpps @ 8°_ 라고 표기되어 있다면 실제 의미는 다음과 같습니다.

> “이 Scanner는 올바르게 튜닝되었을 때 8° 스캔 각도에서 초당 30,000포인트로 ILDA 테스트 패턴을 재현할 수 있다.”

이는 실제 환경 성능을 포괄적이거나 완전히 표준화된 방식으로 측정한 값은 아닙니다. 사실 원래 벤치마크로 설계된 것도 아니며, **튜닝 절차**에 사용되던 것입니다. 정해진 포인트 레이트(예: 초당 30,000포인트)로 알려진 패턴을 실행하고, 올바르게 보일 때까지 댐핑과 게인을 조정하는 방식입니다.

그래도 현재 가장 널리 쓰이는 기준이며, 적어도 신뢰할 수 있는 제조사의 경우 Scanner 품질을 파악하는 데 좋은 참고가 됩니다. 하지만 _덜 신뢰할 수 있는_ 제조사의 경우에는...

#### 정격 사양대로 Scanner를 테스트하고 싶다면

{% hint style="danger" %}
**이것은 고급 기법이며, 주의하지 않으면 Scanner를 손상시킬 수 있습니다. 자신이 무엇을 하는지 정확히 아는 경우가 아니라면 권장하지 않습니다.**
{% endhint %}

[ILDA Test Pattern](https://ilda.com/technical.htm?r=7950)을 출력할 수 있는 소프트웨어가 필요합니다. 제 생각에는 LaserShowGen이 가능할 수도 있습니다. 그리고 출력 크기를 지정된 스캔 각도(예: 8°)에 맞게 조정해야 합니다. 출력 분석 방법은 ILDA 문서를 참고하세요.

#### 좋은 벤치마크가 아닐 수 있는 이유

우선 Scanner가 좋아도, 해당 테스트 패턴에 최적화된 방식으로 튜닝되어 있지 않다면 패턴이 잘못 보일 수 있습니다.

“좋은” Scanner와 “나쁜” Scanner를 대략적으로 구분하는 데는 유용할 수 있지만, 제조사가 이 사양을 다소 느슨하게 사용하거나 유리하게 해석하는 경우도 있습니다.

#### 그렇다면 좋은 Scanner는 어떻게 고르면 될까요?

저는 신뢰할 수 있는 제조사의 제품인지 확인하는 것이 중요하다고 생각합니다. Cambridge Technology (CT), Eye Magic (EMS), ScannerMAX(Pangolin의 자회사) 같은 고가의 하이엔드 Scanner 제조사는 항상 좋은 선택이고 크게 실패할 일이 없습니다. 하지만 Scanner 한 쌍이 약 $1000 정도라면, 처음 시작하는 많은 사람에게는 레이저보다 더 비싼 경우도 있습니다!

그래서 저는 주로 중국 제조사의 제품을 사용해 왔습니다. Dragon Tiger (DT) Scanner는 괜찮고 가격도 합리적이며, LightSpace도 이를 사용하는 것으로 알고 있습니다. 다른 많은 제조사(일부 모델의 OPT와 Able 포함)도 DT 기반 시스템을 사용합니다.

Phenix Technology (PT)는 일반적으로 한 단계 낮은 등급이지만, 솔직히 대부분의 용도에는 충분할 가능성이 큽니다.

**Scanner에 브랜드가 없다면, 그때는 품질을 걱정해야 할 가능성이 큽니다!**

#### Liberation이 도움이 되는 방식

우선 대부분의 작업에는 아주 비싼 Scanner가 필요하지 않습니다! 합리적인 가격의 30kpps DT, 또는 PT도 괜찮습니다. 기본 Scanner 설정은 의도적으로 보수적으로 되어 있으며, 대부분의 경우 _조정할 필요가 없습니다_ (_Scanner sync_ 제외).

더 좋은 Scanner를 사용하더라도 필요 이상으로 강하게 구동할 이유는 없습니다. 이렇게 하면 수명을 크게 늘릴 수 있습니다.

#### "포인트 스트림"이란?

아마 이 용어를 들어본 적이 있을 것입니다. Scanner의 경로를 제어하는 방식입니다.

포인트 스트림은 X/Y 위치 목록이며, 각 위치에는 색상이 포함됩니다. 흰색 선 같은 것을 그리려면 해당 선을 따라 일련의 포인트를 보내고, 모든 포인트를 흰색으로 설정합니다. 그러면 Scanner는 고정된 속도, 즉 초당 포인트 수(PPS)에 따라 포인트에서 포인트로 이동하고, 빔이 그 형태를 따라가며 그립니다.

#### 기존 레이저 소프트웨어에서의 동작 방식

기존 레이저 소프트웨어는 각 cue마다 포인트 스트림을 저장합니다. 애니메이션 cue의 경우 일반적으로 각 프레임마다 별도의 포인트 스트림이 있다는 뜻입니다. 핵심은 모든 것이 완전히 미리 정해져 있다는 점입니다. 따라서 포인트 레이트를 높이면 Scanner가 동일한 사전 정의 데이터 안에서 더 빠르게 이동합니다.

{% hint style="info" %}
예전 소프트웨어에서는 이 방식이 필요했습니다. 컴퓨터가 실시간으로 포인트 스트림을 생성할 만큼 충분히 빠르지 않았기 때문입니다. 그래서 보통 별도의 cue editor가 있고, 이를 사용해 애니메이션의 각 프레임 데이터를 미리 생성했습니다.

콘텐츠가 기가바이트 단위의 공간을 차지할 수 있는 이유도 여기에 있습니다. 사실상 높은 샘플 레이트의 크고 압축되지 않은 파형을 저장하는 것과 같습니다.
{% endhint %}

#### Liberation에서 "point rate"의 의미가 덜 중요한 이유

Liberation은 포인트 스트림을 실시간으로 생성하므로 매우 큰 유연성을 제공합니다. Laser Settings 패널의 "Scanner speed" 설정을 확인해 보세요. 이 설정을 통해 Scanner를 빠르게 또는 느리게 만들 수 있지만, 중요한 점은 기본 포인트 레이트(PPS)를 변경하지 않는다는 것입니다.

#### 잠깐, 뭐라고요? 어떻게요?

처음에는 이상하게 들린다는 것을 압니다.

Liberation은 포인트 스트림을 실시간으로 생성하기 때문에 해당 포인트 스트림을 조정할 수 있습니다. 포인트 간격이 넓을수록 Scanner는 더 빠르게 움직입니다. 포인트가 서로 가까울수록 Scanner는 더 느리게 움직입니다.

{% hint style="info" %}
최근 버전의 Liberation에서는 실제 _point rate_(advanced settings에 있음)가 Scanner 속도에 전혀 영향을 주지 않습니다. 대신 렌더러가 선택한 포인트 레이트에 맞도록 포인트 분포를 조정하면서, Scanner의 움직임은 그대로 유지합니다.

이는 포인트 경로의 "resolution"에는 영향을 주지만, 주로 그래픽에서 차이가 나며(_scanner sync_ 설정을 더 세밀하게 조정하는 데 도움이 될 수 있습니다).
{% endhint %}

#### 좋습니다! 그래서 이게 실제로 무엇을 의미하나요?

좋은 질문입니다. 제 팁은 다음과 같습니다.

* 브랜드가 없는 Scanner는 피하세요. 더 빠른 Scanner를 구할 수 있다면 그렇게 하세요. 최소 30KPPS를 권장합니다.
* 대부분의 경우 DT30 Scanner면 충분하며, 저가 레이저에서는 PT30 Scanner도 아마 괜찮을 것입니다.
* 그래픽 작업을 한다면, 대부분의 경우 더 빠른 Scanner보다 더 많은 레이저가 더 효과적입니다.
* 하이엔드 구성으로 올라가면, 잘 알려진 하이엔드 브랜드라면 어느 것이든 괜찮습니다.
* 가장 저렴한 무브랜드 Scanner밖에 구할 수 없다면, Liberation의 기본 설정은 꽤 보수적이므로 기본적인 빔 작업에는 아마 괜찮은 결과를 얻을 수 있습니다. 부담이 있어 보이면 **Speed** 설정을 낮추세요(단, 포인트 레이트는 변경하지 마세요!).

#### 그리고 ILDA Test Pattern은?

…여전히 캘리브레이션과 기준 도구로 매우 유용하지만, 포괄적인 벤치마크로 설계된 것은 아니며 제조사에 의해 잘못 사용되거나 느슨하게 해석될 수 있습니다.
