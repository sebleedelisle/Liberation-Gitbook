# 🟩 Введение в Clip Editor

Clip Editor — универсальный способ создавать лазерный контент и одна из ключевых частей Liberation. В нём легко делать простые вещи, но при этом он достаточно гибкий для создания очень сложных и продвинутых эффектов.

{% hint style="info" %}
Редактор на основе nodes был самой первой частью Liberation! Идея появилась после разговора с Робом Стэнли на встрече лазерщиков в Великобритании в 2018 году — мы обсуждали, как мог бы выглядеть «объектно-ориентированный» дизайнер лазерного контента.

Хотя на первый взгляд всё кажется простым, построить такую систему довольно сложно. Но к началу 2019 года у меня уже была рабочая демоверсия, которая подтвердила концепцию, — и с неё начался весь этот путь!
{% endhint %}

Это визуальный редактор на основе nodes, или [архитектура node-графа](https://en.wikipedia.org/wiki/Node_graph_architecture#Node_graph), которая будет знакома тем, кто работал с TouchDesigner, MaxMSP или VVVV. При этом Clip Editor немного отличается и устроен несколько проще, потому что он разработан специально для векторной графики.

Открыть Clip Editor можно, щёлкнув правой кнопкой мыши по кнопке Clip и выбрав _EDIT CLIP_. Либо щёлкните правой кнопкой мыши по пустой кнопке Clip и выберите _CREATE AND EDIT CLIP_.

### Обзор

В Clip Editor вы увидите:

* Кнопки **Creator** и **Operator node** в верхней части
* Кнопки **Oscillator node** слева
* Предпросмотр контента в панели справа; если щёлкнуть по node, появится второй предпросмотр, показывающий контент непосредственно на этом node.
* Все nodes и соединения для Clip, который вы редактируете (если это новый Clip, область будет пустой)
* Панель Clip Editor с различными параметрами

Во время редактирования вы также будете видеть, как Clip выглядит в 3D-визуализаторе на фоне.

{% hint style="info" %}
Если в 3D-визуализаторе нет Output, возможно, нужно включить нужные zones с помощью кнопок zone. Также убедитесь, что включён параметр _Preview to lasers_; подробнее см. [Панель Clip Editor](clip-editor-intro.md#clip-editor-panel) ниже.
{% endhint %}

### Создание Clip

Обычно вы начинаете с одного или нескольких [nodes Creator](creator-nodes.md), а затем подключаете [Operator nodes](operator-nodes/) слева направо, чтобы обрабатывать контент. Когда вы перемещаете Creators и/или operators рядом друг с другом, они автоматически соединяются. Чтобы снова разъединить их, просто перетащите их в стороны.

### Добавление nodes в Clip

Нажмите и перетащите одну из кнопок node сверху или слева в пустое место внутри Clip Editor.

### Настройка параметров node

Нажмите кнопку со значком шестерёнки в правом верхнем углу node, чтобы открыть панель свойств для этого node.

### Включение и отключение node

Нажмите кнопку со значком питания в левом верхнем углу node, чтобы включить или отключить этот node. Node станет менее ярким, показывая, что сейчас он не активен. Обратите внимание: контент всё равно проходит через operator, даже если он отключён, но сам node не влияет на контент.

### Соединение nodes

Контент создаётся с помощью node Creator и передаётся по nodes слева направо; каждый operator каким-либо образом влияет на контент и передаёт его следующему operator. Всё, что остаётся в конце цепочки, становится контентом Clip. Creators и Operators автоматически соединяются, когда вы перемещаете их близко друг к другу. Чтобы разъединить их, снова перетащите их в стороны.

{% hint style="info" %}
К входу следующего node можно подключить больше одного node. Это удобно, когда нужно объединить несколько фрагментов контента.
{% endhint %}

### Свойства и сокеты node

В нижней части каждого node расположен набор сокетов; каждый сокет соответствует одному свойству внутри node, например яркости, позиции, масштабу, повороту и т. д.

[Node типа Oscillator](oscillators/) можно подключать к этим сокетам снизу и использовать для анимации таких параметров. У Oscillator nodes есть выход в верхней части: нажмите и перетащите, чтобы вытянуть соединение, затем отпустите его на сокете свойства другого node.

### Oscillator nodes

Oscillator nodes используются для изменения свойств с течением времени. Обычно они представляют волновые формы, например пилообразную или синусоидальную волну, но некоторые Oscillator nodes используют внешние входные данные, например уровень сигнала с микрофона.

{% hint style="info" %}
Если вы когда-либо пользовались аналоговым синтезатором, вам знакома концепция осцилляторов: их часто используют для создания волновых форм или изменения звука со временем. Осцилляторы в Liberation выполняют похожую задачу.

**Интересный факт:** название _Liberation_ вдохновлено Moog Liberation — синтезатором-кейтаром, выпущенным в 1980 году и ставшим известным благодаря Херби Хэнкоку, Жану-Мишелю Жарру и даже Джеймсу Брауну!
{% endhint %}

У осцилляторов всегда есть параметры _range_, которые задают минимальное и максимальное значение изменяемого свойства. А у _Wave Oscillators_ всегда есть параметр _duration_, определяющий, насколько быстро осциллятор меняет значение. Подробнее см. [Волновые осцилляторы](oscillators/wave-oscillators.md).

### Панель Clip Editor

* Timer — в верхней части панели отображается текущее время Clip по мере его воспроизведения
* _RETRIGGER_ — перезапускает Clip с начала; особенно полезно, если Clip не зациклен
* _Preview to lasers_ — когда этот параметр включён, 3D-визуализатор обновляется по мере редактирования Clip. Если отключить его, вы будете видеть те Clips, которые запущены вне редактора. Это глобальная настройка, а не настройка отдельного Clip.
* _UNDO/REDO_ — отмена и повтор действий в самом Clip Editor. Также назначено на `Cmd / Ctrl + Z` и `Cmd / Ctrl + Shift + Z`.
* _SAVE CLIP_ — сохраняет изменения, но предупреждает о перезаписи
* _SAVE AS A COPY_ — сохраняет Clip в следующий свободный слот в Clip Deck. Это становится новой позицией для Clip, и все последующие сохранения будут выполняться уже в этом месте.
* _EXIT EDITOR_ — закрывает Clip Editor. Если есть несохранённые изменения, появится панель подтверждения.
