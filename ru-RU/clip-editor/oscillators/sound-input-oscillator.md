# ✅ Осциллятор Sound input

## <img src="../../.gitbook/assets/sound-input-icon.png" alt="" data-size="line"> Sound input

Преобразует уровень входного звукового сигнала в значение свойства.

{% hint style="info" %}
Осциллятор Sound input использует звуковой интерфейс по умолчанию, но его можно изменить в _Preferences_. Откройте меню _Liberation -> Preferences._

В настройках _Sound Input_ можно выбрать звуковой интерфейс, который вы хотите использовать, а также настроить несколько дополнительных параметров для регулировки уровня громкости.
{% endhint %}

* **range min / range max** — минимальное и максимальное значения, на которые отображается форма сигнала
* **channel** — если у вашего звукового интерфейса больше одного входного канала, здесь можно выбрать, с какого канала считывать сигнал.

{% hint style="info" %}
Интересный прием — подать несколько звуковых сигналов с микшерного пульта. Так разные Clips смогут реагировать на разные музыкальные инструменты.
{% endhint %}

{% hint style="info" %}
Во всех _Sound Inputs_ одновременно можно использовать только один звуковой интерфейс (выбирается на панели _App Settings_). Если для этого нужно использовать больше одного интерфейса, в macOS можно [создать агрегатное устройство](https://support.apple.com/en-gb/HT202000), чтобы объединить входы в один виртуальный источник звука. (В Windows тоже есть много приложений, которые это делают, но я их не пробовал).
{% endhint %}

* **clamp min / clamp max** — используйте эти параметры, чтобы выбрать диапазон уровней, на который нужно реагировать. Если gate и limit (на панели _App Settings_) настроены правильно, обычно менять это не требуется, но эти параметры можно использовать для творческих эффектов.

{% hint style="info" %}
Когда у Clip есть осциллятор _Sound Input_, на кнопке Clip появляется небольшой значок микрофона.
