# 🟩 Эффекты

Система эффектов в Liberation — удобный и гибкий способ изменять вывод Clip в реальном времени. Эффекты полностью настраиваются: с их помощью можно заставить всё мигать, вращаться, менять цвета или даже хаотично перемещаться!

Всё, что можно сделать в Clip Editor, можно использовать как эффект. Более того, эффекты редактируются в том же самом редакторе nodes, что и Clips! См. [Редактирование эффектов](effects.md#editing-effects "mention"). Творческие возможности практически безграничны.

По умолчанию кнопки эффектов 1–8 находятся под кнопками zones, а эффекты 9–24 — это маленькие кнопки в нижней части.

#### Применение эффекта

Нажмите кнопку эффекта, чтобы включить или выключить эффект. Ещё лучше — используйте слайдеры 1–8 на APC40, чтобы плавно вводить и выводить эффекты. Чтобы плавно ввести эффект без APC40, нажмите кнопку и перетащите вверх или вниз. Также можно щёлкнуть правой кнопкой по кнопке эффекта и настроить слайдер _level_.

{% hint style="warning" %}
Нажатие кнопки эффекта сразу активирует этот эффект. Но учтите: если _level_ установлен на ноль, ничего не произойдёт! Нажмите и перетащите кнопку, чтобы изменить уровень, щёлкните правой кнопкой и используйте слайдер _level_ или воспользуйтесь фейдерами APC40.
{% endhint %}

#### Эффекты и задержка zone в Clip

Эффекты наследуют настройку задержки zone для каждого Clip, который сейчас запущен. Поэтому если в вашем Clip есть задержка, движущаяся слева направо, и вы добавляете эффект мигания, это мигание тоже будет задерживаться слева направо.

{% hint style="info" %}
То, как эффекты наследуют задержку zone из Clip, очень сложно описать словами, но это сразу становится понятно, когда вы попробуете сами!

На мой взгляд, это один из самых интересных и творческих инструментов, встроенных в Liberation. Попробуйте — и вы поймёте, о чём я!
{% endhint %}

#### Параметры эффектов

Добавьте параметр в свой эффект с помощью _Parameter node._ Система Parameter позволяет изменять несколько настроек внутри эффекта извне. Подробнее см. [Управление параметрами](clip-editor/oscillators/parameter-control.md "mention").

Используйте поворотные регуляторы 1–8, чтобы настраивать _parameter_ для каждого эффекта. Также можно щёлкнуть правой кнопкой по кнопке эффекта и настроить слайдеры параметров. Изменение параметра выполняет разные действия в зависимости от того, как настроен эффект. Ниже приведён список эффектов по умолчанию и описание того, что делают их параметры.

{% hint style="info" %}
Поворотные регуляторы 1–8 находятся в верхней части APC40 Mk2 и справа сверху на Mk1. См. также: [Справочник по APC40](reference/apc40-reference.md "mention")
{% endhint %}

{% hint style="info" %}
Маленькие числа на кнопках эффектов показывают _level_ и _parameter_ эффекта. _level_ управляется фейдером на APC40; также можно нажать кнопку и перетащить её. Параметр настраивается поворотными регуляторами APC40; также можно щёлкнуть правой кнопкой и настроить его мышью.
{% endhint %}

#### Эффекты по умолчанию

<figure><img src=".gitbook/assets/qs-default-effects.png" alt=""><figcaption></figcaption></figure>

1. **Randomiser** :\
   Применяет к выводу Clip хаотичное движение. Параметр регулирует величину и скорость хаоса.
2. **Sine wave** :\
   Искажает всё содержимое по движущейся синусоиде. Параметр регулирует длину волны.
3. **Rotation** :\
   Вращает всё содержимое. Параметр регулирует скорость вращения.
4. **Horizontal flip** :\
   Сжимает и растягивает всё содержимое по горизонтали. Параметр регулирует скорость.
5. **Scale** :\
   Постоянно масштабирует всё содержимое от полного размера до нуля. Параметр регулирует скорость.
6. **Hue** :\
Изменяет оттенок всего содержимого, но не меняет насыщенность (т. е. всё белое остаётся белым). Параметр регулирует оттенок.
7. **Saturation and hue** :\
Изменяет оттенок всего содержимого и также полностью насыщает цвет (т. е. всё белое меняется на выбранный цвет). Параметр регулирует оттенок.
8. **Flash** :\
   Постоянно меняет яркость всего содержимого от полной до нулевой, создавая мигание. Параметр регулирует скорость мигания.

В нижнем ряду есть ещё 16 цветовых эффектов, которые применяют заранее заданные значения оттенка и насыщенности.

Обратите внимание: это эффекты по умолчанию, но их можно отредактировать так, чтобы они делали почти всё, что вам нужно!

### Apply to groups

Можно выбрать, на какие группы будет влиять эффект. Щёлкните правой кнопкой и переключите флажки групп с подписью _Apply to groups._

Я в основном использую такую настройку, когда работаю с графикой Canvas и лазерными лучами отдельно. Я назначаю все canvas Clips группе 5, а затем исключаю эту группу из эффектов, которые не должны влиять на эти графические Clips.

Также это можно использовать, чтобы в реальном времени применить 2 разных изменения цвета к 2 группам лазеров одновременно. Создайте два эффекта изменения цвета и выберите, к каким группам Clips применяется каждый из них.

### MX group

Сокращение от _Mutually Exclusive_. Это способ объединить эффекты в группу так, чтобы одновременно мог быть активен только один эффект из этой группы. Обратите внимание, что из эффектов изменения цвета по умолчанию одновременно может быть активен только один. Это потому, что все они находятся в MX Group 1.

Эта функция отключена, если настройка _MX Group_ равна 0.

### Редактирование эффектов

Щёлкните правой кнопкой по любому эффекту и нажмите кнопку _EDIT EFFECT_, чтобы открыть редактор эффектов. Обратите внимание, что этот редактор идентичен Clip Editor!

Редактируйте эффект так же, как редактировали бы любой Clip. См. [Clip Editor](clip-editor/ "mention").

В эффекте должен быть как минимум один node Creator. Это может быть что угодно (линия, круг, фигура, даже текст!), но лучше выбрать то, что наиболее понятно отображается в предпросмотре на кнопке эффекта.

Когда эффекты применяются, все nodes Creator в эффекте заменяются выводом Clips, которые сейчас запущены.

{% hint style="warning" %}
По крайне скучным техническим причинам nodes "trails" не работают внутри эффекта. То же относится к настройке "delay" внутри pattern nodes (они используют ту же систему). Это будет исправлено в будущих версиях.
{% endhint %}
